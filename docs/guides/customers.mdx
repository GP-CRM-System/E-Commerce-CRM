# Customer Management

Customers are the core entities in Briefly CRM. Each customer can have:

- **Basic Info**: Name, email, phone
- **Address**: City, country, postal code
- **Source**: Origin of the customer (manual, import, shopify, etc.)
- **RFM Scoring**: Recency, Frequency, Monetary analysis
- **Lifecycle Stage**: Dynamic stage based on order behavior
- **Churn Risk**: Score 0-100
- **VIP Status**: Top 10% by lifetime value

## Endpoints

### Customer CRUD

| Method   | Endpoint         | Description     |
| -------- | ---------------- | --------------- |
| `GET`    | `/customers`     | List customers  |
| `POST`   | `/customers`     | Create customer |
| `GET`    | `/customers/:id` | Get customer    |
| `PUT`    | `/customers/:id` | Update customer |
| `DELETE` | `/customers/:id` | Delete customer |

### Analytics

| Method | Endpoint                       | Description              |
| ------ | ------------------------------ | ------------------------ |
| `GET`  | `/customers/:id/analytics`     | Get RFM scores & metrics |

## Query Parameters

| Parameter        | Type    | Description                        |
| ---------------- | ------- | ---------------------------------- |
| `page`           | integer | Page number                        |
| `limit`          | integer | Items per page                     |
| `sortBy`         | string  | Sort field                         |
| `sortOrder`      | string  | asc or desc                        |
| `search`         | string  | Search by name/email               |
| `city`           | string  | Filter by city                     |
| `source`         | string  | Filter by source                   |
| `segmentId`      | string  | Filter by segment                  |
| `lifecycleStage` | string  | Filter by stage                    |
| `churnRisk`      | string  | Filter by risk (low, medium, high) |
| `isVip`          | boolean | Filter by VIP status               |

## Customer Events

Track customer activities with custom events:

```json
{
    "type": "order_placed",
    "metadata": {
        "orderId": "ord_123",
        "amount": 99.99
    }
}
```

**Event Types:**

- `order_placed` - Order created
- `refund_issued` - Refund processed
- `tag_added` - Tag applied
- `tag_removed` - Tag removed
- `note_added` - Note created
- `lifecycle_changed` - Stage changed

## Lifecycle Stages

Customers automatically move through stages based on order behavior:

| Stage         | Criteria                              |
| ------------- | ------------------------------------- |
| **PROSPECT**  | New customer, no orders               |
| **LEAD**      | First order placed                    |
| **ONE_TIME**  | Exactly 1 order                       |
| **RETURNING** | 2-4 orders                            |
| **LOYAL**     | 5+ orders with consistent frequency   |
| **VIP**       | Top 10% by lifetime value             |
| **AT_RISK**   | No orders in 90+ days                 |
| **CHURNED**   | No orders in 180+ days                |
| **WINBACK**   | Churned customer who placed new order |

Run `/cron/lifecycle` to recalculate all stages.

## Churn Risk

Churn risk is calculated from:

- Days since last order
- Order frequency decline
- Average order value trend
- RFM recency score

Risk levels:

- **Low**: Score 0-33
- **Medium**: Score 34-66
- **High**: Score 67-100
