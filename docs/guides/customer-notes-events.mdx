# Customer Notes & Events

Track customer interactions through notes and custom events.

## Notes

Notes are timestamped text entries attached to customers for tracking conversations, feedback, or important information.

### Endpoints

| Method   | Endpoint                       | Description |
| -------- | ------------------------------ | ----------- |
| `GET`    | `/customers/:id/notes`         | List notes  |
| `POST`   | `/customers/:id/notes`         | Create note |
| `PATCH`  | `/customers/:id/notes/:noteId` | Update note |
| `DELETE` | `/customers/:id/notes/:noteId` | Delete note |

### Create Note

```http
POST /customers/:id/notes
```

**Body:**

```json
{
    "content": "Customer called about order status"
}
```

**Response:**

```json
{
    "message": "Note created successfully",
    "data": {
        "id": "note_xxx",
        "content": "Customer called about order status",
        "createdBy": "usr_xxx",
        "createdAt": "2024-01-15T10:30:00Z",
        "updatedAt": "2024-01-15T10:30:00Z"
    },
    "timestamp": "2024-01-15T10:30:00Z",
    "path": "/api/customers/cus_xxx/notes"
}
```

### List Notes

```http
GET /customers/:id/notes
```

**Query Parameters:**

| Parameter   | Type    | Description            |
| ----------- | ------- | ---------------------- |
| `page`      | integer | Page number            |
| `limit`     | integer | Items per page         |
| `sortBy`    | string  | Sort field (createdAt) |
| `sortOrder` | string  | asc or desc            |

### Update Note

```http
PATCH /customers/:id/notes/:noteId
```

```json
{
    "content": "Updated note content"
}
```

### Delete Note

```http
DELETE /customers/:id/notes/:noteId
```

## Events

Customer events track actions like orders, refunds, and tag changes. They're useful for building customer timelines and triggering automated workflows.

### Endpoints

| Method   | Endpoint                         | Description  |
| -------- | -------------------------------- | ------------ |
| `GET`    | `/customers/:id/events`          | List events  |
| `POST`   | `/customers/:id/events`          | Create event |
| `PATCH`  | `/customers/:id/events/:eventId` | Update event |
| `DELETE` | `/customers/:id/events/:eventId` | Delete event |

### Event Types

| Event Type          | Description      |
| ------------------- | ---------------- |
| `order_placed`      | Order created    |
| `refund_issued`     | Refund processed |
| `tag_added`         | Tag applied      |
| `tag_removed`       | Tag removed      |
| `note_added`        | Note created     |
| `lifecycle_changed` | Stage changed    |

### Create Event

```http
POST /customers/:id/events
```

**Body:**

```json
{
    "type": "order_placed",
    "metadata": {
        "orderId": "ord_123",
        "amount": 99.99
    }
}
```

- `type` - Type of event (required)
- `metadata` - Additional event data (optional)

**Response:**

```json
{
    "message": "Event created successfully",
    "data": {
        "id": "evt_xxx",
        "type": "order_placed",
        "metadata": {
            "orderId": "ord_123",
            "amount": 99.99
        },
        "createdBy": "usr_xxx",
        "createdAt": "2024-01-15T10:30:00Z"
    },
    "timestamp": "2024-01-15T10:30:00Z",
    "path": "/api/customers/cus_xxx/events"
}
```

### List Events

```http
GET /customers/:id/events
```

**Query Parameters:**

| Parameter   | Type    | Description            |
| ----------- | ------- | ---------------------- |
| `page`      | integer | Page number            |
| `limit`     | integer | Items per page         |
| `type`      | string  | Filter by event type   |
| `sortBy`    | string  | Sort field (createdAt) |
| `sortOrder` | string  | asc or desc            |

### Using Events

Events are useful for:

- Building customer timelines
- Triggering workflows (via webhooks)
- Understanding customer behavior
- Tracking key milestones

For example, listen for `order_placed` events to trigger post-purchase email sequences.
