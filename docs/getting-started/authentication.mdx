# Authentication

Briefly CRM uses [Better Auth](https://www.better-auth.com/) for enterprise-grade authentication.

## Supported Methods

- **Email/Password**: Traditional login with secure password hashing
- **Google OAuth**: Single sign-on with Google accounts
- **API Tokens**: Bearer token authentication for API access

## Role-Based Access Control

The system supports custom roles with granular permissions:

| Resource    | Actions                              |
| ----------- | ------------------------------------ |
| `customers` | `create`, `read`, `update`, `delete` |
| `orders`    | `create`, `read`, `update`, `delete` |
| `products`  | `create`, `read`, `update`, `delete` |
| `imports`   | `create`, `read`                     |
| `exports`   | `create`, `read`                     |
| `roles`     | `manage`                             |

## Getting an Access Token

### Email/Password

```bash
curl -X POST http://localhost:3000/api/auth/sign-in/email \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "your-password"}'
```

### Using the Token

Include the token in Authorization header:

```bash
curl http://localhost:3000/api/customers \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Organization Scoping

All data is scoped to organizations. Users can only access data within their organization unless they have administrative privileges.
