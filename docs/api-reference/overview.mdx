# API Reference

The Briefly CRM API is documented using OpenAPI 3.1. The interactive playground below allows you to explore and test all endpoints.

<Info>
    All endpoints require authentication. Include your Bearer token in requests.
</Info>

## Base URL

```
/api
```

## Available Endpoints

### Customers

| Endpoint                                | Description              |
| --------------------------------------- | ------------------------ |
| `GET /customers`                        | List all customers       |
| `POST /customers`                       | Create a customer        |
| `GET /customers/:id`                    | Get customer by ID       |
| `PATCH /customers/:id`                  | Update customer          |
| `DELETE /customers/:id`                 | Delete customer          |
| `GET /customers/:id/analytics`          | Get RFM scores & metrics |
| `GET /customers/:id/notes`              | List customer notes      |
| `POST /customers/:id/notes`             | Create note              |
| `PATCH /customers/:id/notes/:noteId`    | Update note              |
| `DELETE /customers/:id/notes/:noteId`   | Delete note              |
| `GET /customers/:id/events`             | List customer events     |
| `POST /customers/:id/events`            | Create event             |
| `PATCH /customers/:id/events/:eventId`  | Update event             |
| `DELETE /customers/:id/events/:eventId` | Delete event             |

## Response Format

All responses follow a consistent format:

```json
{
    "message": "Success message",
    "data": {},
    "timestamp": "2024-01-01T00:00:00.000Z",
    "path": "/api/endpoint"
}
```

Paginated responses include:

```json
{
    "success": true,
    "message": "Success message",
    "data": [],
    "pagination": {
        "page": 1,
        "limit": 20,
        "total": 100,
        "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00.000Z",
    "path": "/api/endpoint"
}
```

## Error Responses

```json
{
    "message": "Error description",
    "code": "ERROR_CODE",
    "status": 400,
    "timestamp": "2024-01-01T00:00:00.000Z",
    "path": "/api/endpoint",
    "details": {}
}
```
